# Endpoints
<customspacer />
## `/articles`

Dieser Endpunkt erlaubt die Methode `GET` und besitzt idealerweise die folgende Struktur:
<customspacer />
```json
// json
{
    "id": 0,
    "title": "Der Titel des Artikels",
    "content": "Der Text (kann HTML enthalten, gereinigt)",
    "time": {
        "year": 2024,
        "month": 1,
        "day": 31,
        "hhmm": "16:40"
    },
    "tags": [
        "gewöhnlich leer", "nicht implementiert"
    ],
    "customLink": null, // reserviert, kann aber auch ein string sein
}
```
<customspacer />
## `/documentation<LANG>`

Dieser Endpunkt wird für die Volltextsuche verwendet und beinhaltet alle Erst- und Zweitüberschriften.
<Warning content="Dritte Überschriften sind nicht behinhaltet!" title="Warnung" />

Wenn der Endpunkt mit der Methode `GET` mit folgender URL aufgerufen wird; `https://api.xefili.dev/documentationDE`, wird ein Array mit allen Überschriften und deren Links zurückgegeben.

### Sprachcodes

Dieser Endpunkt ist für mehrere Sprachen maßgeschneidert: englisch (EN) und deutsch (DE). Um auf eine bestimmte Sprache zuzugreifen, einfach den Sprachcode in `<LANG>` einsetzen.
<Tip title="Weitere Sprachen" content="Sollte ich weitere Sprachen hinzufügen, werden die Sprachcodes gemäß ISO 3166-1 alpha-2 implementiert." />
